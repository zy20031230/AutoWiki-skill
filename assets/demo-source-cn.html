MemSkill

Zhang et al., 2026

arXiv 2026-02

MemSkill 将智能体记忆提取从固定的手工设计操作重构为可学习、可演化的技能库 — 控制器选择技能，LLM 执行器生成技能条件化的记忆，设计器从困难案例中演化技能集。

因子（CRGP）

背景 Context

随着 LLM 智能体参与更长、更开放的交互，它们必须处理不断增长的历史记录。大多数记忆系统依赖静态的、手工设计的操作 — 固定原语把对"存什么、怎么改"的人类假设硬编码进去。

相关工作 Related

MemoryOS, A-MEM, MemGPT, MemoryBank：静态提取管道。Memory-R1, Mem-α：用 RL 做记忆管理但原语固定。Evo-Memory, MemEvolve：自演化记忆但不针对技能演化本身。

缺口 Gap

没有先前工作把记忆操作本身视为一等可学习实体。领域混淆了两个正交问题：记什么（记忆内容）和怎么记（记忆技能）。

方案 Proposal

一个包含结构化记忆操作的技能库。控制器（RL）选择 Top-K 技能 → LLM 执行器单次生成技能条件化的记忆 → 设计器从困难案例中演化技能。选择策略和技能集的闭环优化。

批判性分析 — 新颖洞察

记忆操作（"怎么记"）是独立于记忆内容（"存了什么"）的、可单独演化的层 — 将其作为可学习技能解锁了静态系统无法访问的新优化维度。

先前认知领域将记忆管理视为内容问题，用固定的过程逻辑解决 认知更新提取过程本身就是一个变量 — 把"怎么记"和"记什么"分离，可以联合优化两者

跨段（多轮）技能条件化记忆构建优于逐轮交错操作 — 记忆提取的粒度对下游有显著影响。

先前认知逐轮处理被认为是捕获细粒度记忆更新所必需的 认知更新将多轮合并为一个段，让技能能推理跨轮的时间关系，产生更连贯的记忆
根本局限

技能设计器基于困难案例演化技能，但没有机制检测技能冗余或矛盾 — 技能库膨胀和不一致可能随时间增长。

根因：技能按单个困难案例上的效用评估，而非按与全库的一致性评估。
启示：自演化技能库有失控膨胀的风险 — 需要一个维护机制（类似 SkillFoundry 的剪枝/合并）。

RL 控制器为特定交互分布学习技能选择；迁移到新领域时，即使技能可泛化，选择策略也可能次优。

根因：技能选择策略编码了任务特定先验 — 它们不是分布无关的。
启示：泛化结果可能反映的是技能内容泛化而非控制器泛化。

研究前沿

在单一训练目标中联合优化记忆技能内容和技能选择策略。

前置条件：技能内容的可微或 REINFORCE 兼容表示，允许梯度从下游性能回流到设计器。
最接近尝试：Memory-R1 通过 RL 优化记忆但用固定原语 — 将技能内容作为可学习变量是自然延伸。

跨智能体记忆系统的技能库共享 — 让独立训练的 MemSkill 实例通过兼容性注册表共享演化技能。

前置条件：形式化的技能表示，带有类型化的适用性条件以支持自动兼容性检查。
最接近尝试：EvoSkills 展示了任务执行技能的跨模型迁移；记忆技能的类似迁移尚未探索。

时间关系

MemSkill（2026-02）将 CASCADE（2025-12）建立的自演化技能范式应用到记忆领域，与 SkillRL 和 S1-NexusAgent 同时出现。

extends cascade — CASCADE 将技能习得应用于外部工具掌握；MemSkill 将同样的范式应用于内部记忆操作 — 同样的学习原理，不同的目标层
complements skillrl — SkillRL 通过 RL 共演化演化任务执行技能；MemSkill 通过 RL 控制器 + LLM 设计器演化记忆提取技能 — 正交的技能领域，共享的优化模式
contrasts yunjue-agent — Yunjue 演化可执行的 Python 工具原语；MemSkill 演化抽象的记忆操作规格 — 可执行代码 vs. 声明式技能描述
contrasts evoskills — EvoSkills 通过共演化验证生成多文件可执行技能包；MemSkill 通过 RL + LLM 设计器生成结构化描述 — 不同的技能类型和演化机制
可迁移启发

控制器 + 执行器 + 设计器三元组（选择、应用、从失败中演化）是任何自演化系统的干净三组件架构。

跨段 vs. 逐轮处理是一个可泛化的设计选择 — 最优粒度取决于任务，应该学习而非固定。

困难案例触发的演化机制是持续演化的高效替代 — 聚焦在影响最大的失败案例上。

认知转变
2026-04-08 · 来自 ingest 对话

"记什么"和"怎么记"是两个独立的优化问题，先前的记忆系统一直在混淆它们 — 把它们分离开来，为自演化记忆开辟了一个新的研究方向。

开放问题
? 技能库能增长到多大才使 Top-K 选择成为瓶颈 — 是否存在一个最优规模，超过后性能趋于平台期？
? 演化出的技能卡（自然语言描述）能否自动转换为可执行的记忆过程，桥接 MemSkill 的声明式技能和 SkillRL 的可执行启发式？
? MemSkill 如何与基础 LLM 的上下文学习能力交互 — 更好的基础模型是否需要更少的演化技能？
? 触发设计器演化的最低困难案例率是多少，相对于 LLM 技能精炼的计算成本？
